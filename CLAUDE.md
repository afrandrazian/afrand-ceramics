# Afrand Ceramics

Handmade ceramics shop. Two storefronts (Etsy + personal website), one inventory, auto-synced. The website gives the shop owner brand control and lower fees (Stripe 2.9% vs Etsy ~10-12%).

For full technical context, platform comparisons, and fee breakdowns: see `docs/context.md`

## Who Works Here

- **Shop owner** — content, branding, design, products, business decisions
- **Cowboy developer** — sync scripts, deploy pipeline, automation

Both use Claude Code. Everything is git-tracked — experiment freely.

## How It Deploys

`main` branch = production (once Cloudflare Pages is connected).
Push to main → auto-deploys to afrandceramics.com.
Push to any other branch → Cloudflare generates a preview URL.

Use branches + PRs for changes you want reviewed before going live.
Direct pushes to main go live immediately.

## How the Sync Works

**GitHub Actions** (`.github/workflows/sync-etsy.yml`):
- Pulls listings from Etsy API → writes `products.json` → commits as "Etsy Sync Bot"
- Runs every 30 min (schedule commented out — uncomment after OAuth setup is complete)
- Auto-rotates OAuth refresh tokens (Etsy tokens are single-use)
- `products.json` is auto-generated — manual edits get overwritten on next sync

**overrides.json**:
- Merges on top of `products.json` at render time
- Key = Etsy listing ID (string). Example: `{ "12345": { "hide": true } }`
- Never overwritten by the bot — safe to edit anytime

**Secrets needed** (configure after API approval):
`ETSY_API_KEY`, `ETSY_REFRESH_TOKEN`, `ETSY_SHOP_ID`, `GH_PAT`

### Step 1: Run the OAuth setup

This opens a browser window — the shop owner must sign in to Etsy and click "Allow access."

```bash
ETSY_API_KEY=wz96kb2z2r7buag82tq0br06 node setup/oauth-setup.js
```

What happens:
1. A local server starts on port 3003
2. Browser opens to Etsy's authorization page
3. After clicking "Allow", Etsy redirects back to localhost
4. Terminal prints the **refresh token** and **user ID**

Save the refresh token — you'll need it for the next step.

The script also prints a curl command to find the **shop ID**. Run it and note the shop ID.

**Troubleshooting:**
- "Port 3003 in use" → kill whatever's using it: `lsof -ti:3003 | xargs kill`
- "Node not found" → install Node.js: `brew install node`
- Browser doesn't open → copy the Auth URL from the terminal and paste it manually

### Step 3: Test the sync locally


Check that `products.json` now has real Etsy listings (not example data). The refresh token in `.env` is now stale — the sync outputs a new one. Update `.env` with it.

### Step 4: Set GitHub Secrets

These let GitHub Actions run the sync automatically. Run from the repo directory:

```bash
gh secret set ETSY_API_KEY --body "wz96kb2z2r7buag82tq0br06"
gh secret set ETSY_REFRESH_TOKEN --body "<your refresh token from step 3>"
gh secret set ETSY_SHOP_ID --body "<your shop ID from step 1>"
```

**Note:** There's also a `GH_PAT` secret needed for auto-rotating the refresh token. This requires a GitHub Personal Access Token with `repo` + `secrets` scopes. Ask Mark (the repo owner) to set this up if you don't have permission.

### Step 5: Enable the sync schedule

In `.github/workflows/sync-etsy.yml`, uncomment the schedule lines:

```yaml
on:
  schedule:
    - cron: '*/30 * * * *'
  workflow_dispatch: {}
```

(Remove the `#` from the `schedule:` and `- cron:` lines.)

### Step 6: Test the GitHub Actions workflow

```bash
gh workflow run sync-etsy.yml
gh run list -L 1
```

Watch for a green check. If it fails, check logs with `gh run view <run-id> --log`.
